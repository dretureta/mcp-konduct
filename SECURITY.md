# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.6.x  | :white_check_mark: |
| < 1.6   | :x:              |

Only the latest minor version receives security updates.

## Reporting a Vulnerability

**Do NOT open a public GitHub issue for security vulnerabilities.**

If you discover a security issue, please report it responsibly:

1. **Email**: Send details to the repository maintainer
2. **GitHub Security Advisory**: Use [GitHub's security advisories](https://github.com/dretureta/mcp-konduct/security/advisories)

Please include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Any suggested fixes (optional)

## Response Timeline

- **Acknowledgment**: Within 48 hours
- **Initial Assessment**: Within 1 week
- **Fix Timeline**: Depends on severity — critical issues prioritized

## Scope

Security updates apply to:
- SQL injection vulnerabilities
- Authentication/authorization bypasses
- Data exposure
- Denial of service vulnerabilities
- Any other security-relevant bugs

## Out of Scope

- Issues fixed by updating dependencies (use Dependabot)
- Social engineering attacks
- Physical security
- Client-side vulnerabilities (browser, OS)

## Public Discussion

Do NOT discuss security issues publicly until a fix is released.
